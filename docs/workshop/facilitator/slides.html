Agentic engineering
nine habits

You just watched the loop work once.
These habits are why it worked - and what to do when it doesn't.

Habits · #1 #2 #9

Plan first. Human decides.

  • #1 Human plans, agent executes - review plans, not 400-line diffs
  • #2 Fresh session per ticket - old threads pollute context
  • #9 Decisions stay human - unmade decision in the plan? Stop. Decide. Write it down.

You watched me push back on a correct plan.That reflex is the job.

Habits · #3 · the money slide

Two strikes, then reset

  • A failing thread is sunk cost
  • The context window now contains a record of failure - and it steers the next attempt
  • Kill it. Restart with a tighter plan: name the file, forbid the failed approach, shrink the task

"If you're negotiating with the chat - reset."

Habits · #4

Small, scoped tasks

  • One ticket = one branch = one PR
  • Plan longer than ~5 steps → split the ticket, not the difference
  • Invented scope is where hallucinated requirements live

Today's tickets are pre-scoped. Notice how that feels -
then blame your backlog, not your model.

Habits · #5 #6 #8

Harness engineering

copilot-instructions.md  policy          docs/adr/       decisions
instructions/*.md        style per path  tests/          sub-second, deterministic
AGENTS.md                tools + ritual  tickets         exact repro commands
.github/skills/          procedures
  • Typed the same instruction twice? It belongs in a file
  • Prompts are session-scoped. The harness compounds.

Everything the agent did well just now,
a file taught it.

Habits · #7

Green ≠ correct

  • This suite is green. The code has three live bugs.
  • Run the repro yourself
  • Make the agent prove the regression test fails on the old code
  • Read every hunk of the diff

Verify like your name is on the commit.
It is.
